📱 App Installation (PWA)  ·  Cài đặt ứng dụng
EN
App icon created for all platforms
The "Add to Home Screen" prompt was silently blocked because the app had no icon files. We generated icons in every required size (192 px, 512 px, 180 px for iOS) so Chrome, Safari and Firefox can now show the install prompt on Android, iPhone and desktop.
VN
Tạo icon ứng dụng cho mọi nền tảng
Trình duyệt không hiển thị nút "Thêm vào màn hình chính" vì không có file icon. Chúng tôi đã tạo icon đủ kích thước (192 px, 512 px, 180 px cho iOS) để Chrome, Safari và Firefox có thể hiện nút cài đặt trên Android, iPhone và máy tính.
✅ Done
EN
App orientation fixed
The app was incorrectly configured to launch in landscape-only mode, causing it to open sideways on phones. Fixed to allow natural device orientation.
VN
Sửa lỗi hướng màn hình
Ứng dụng bị cấu hình sai, luôn mở ngang dù điện thoại đang dọc. Đã sửa để ứng dụng tự điều chỉnh theo hướng tự nhiên của thiết bị.
✅ Done
EN
Cache auto-updates on every deploy
Previously, after a new version was deployed users could still see old cached pages until a developer manually bumped a version string. Now the app automatically refreshes its cache whenever a new build goes live on Vercel.
VN
Cache tự động làm mới sau mỗi lần cập nhật
Trước đây, sau khi triển khai phiên bản mới, người dùng vẫn có thể thấy trang cũ cho đến khi lập trình viên cập nhật thủ công. Bây giờ ứng dụng tự động xóa cache cũ mỗi khi có bản build mới trên Vercel.
✅ Done
🔒 Security  ·  Bảo mật
EN
Browser security headers added
Added standard HTTP headers that instruct browsers to block clickjacking, MIME-type sniffing, and unnecessary data leaks. This hardens the app against common web attacks without changing anything the user can see.
VN
Thêm các tiêu đề bảo mật cho trình duyệt
Bổ sung các HTTP header tiêu chuẩn giúp trình duyệt chặn clickjacking, rò rỉ dữ liệu và các tấn công web phổ biến. Người dùng không thấy thay đổi gì, nhưng ứng dụng an toàn hơn đáng kể.
✅ Done
EN
Login session tokens now rotate on every use
When a parent's short-lived login token expired and the system silently renewed it, the long-lived "refresh" token was reused unchanged — meaning that if someone ever stole it they could use it indefinitely. Now both tokens are replaced together on every renewal, closing that window.
VN
Token đăng nhập được thay mới sau mỗi lần làm mới
Trước đây khi token ngắn hạn hết hạn và được gia hạn tự động, token dài hạn ("refresh token") vẫn giữ nguyên — nếu bị đánh cắp có thể dùng mãi mãi. Bây giờ cả hai token đều được thay mới cùng lúc, vô hiệu hóa token cũ.
✅ Done
EN
Kid unlock lockout timer visible on screen
After too many wrong pattern attempts, the child sees a countdown showing exactly how many seconds remain before they can try again, instead of a confusing generic error message.
VN
Hiển thị đồng hồ đếm ngược khi khóa nhập mã
Khi nhập sai mã mở khóa quá nhiều lần, bé sẽ thấy rõ thời gian còn lại (đếm ngược giây) thay vì thông báo lỗi chung chung khó hiểu.
✅ Done
🏆 Badges & Rewards  ·  Huy hiệu & Phần thưởng
EN
"First Login" badge now triggers on real first use
The 🌟 "Chào bạn mới!" badge was only being awarded in the database seed script, not when a child actually unlocked the app for the first time. It is now awarded automatically the very first time the child enters their unlock pattern successfully.
VN
Huy hiệu "Chào bạn mới!" được trao khi lần đầu mở app
Huy hiệu 🌟 này trước đây chỉ được trao qua script khởi tạo dữ liệu, không phải khi bé thực sự dùng app lần đầu. Bây giờ huy hiệu được trao tự động ngay khi bé nhập đúng mã mở khóa lần đầu tiên.
✅ Done
EN
Grade-based badges now awarded automatically
Four new badge triggers added: 🧮 Siêu Toán (Math excellence), 📚 Sao Đọc Sách (Vietnamese excellence), 💯 Điểm 10! (perfect score), and 🎯 Toàn diện (all subjects at "Good" or above). These are checked every time a parent saves a grade.
VN
Huy hiệu theo điểm số được trao tự động
Thêm 4 điều kiện trao huy hiệu mới: 🧮 Siêu Toán (xuất sắc môn Toán), 📚 Sao Đọc Sách (xuất sắc Tiếng Việt), 💯 Điểm 10! (đạt điểm tuyệt đối), và 🎯 Toàn diện (tất cả môn đều từ Giỏi trở lên). Tự động kiểm tra mỗi khi phụ huynh lưu điểm.
✅ Done
EN
Points awarded when marking homework done
Completing homework now correctly adds points to the child's total and records a streak day, in addition to logging the activity event that was already in place.

Cộng điểm khi hoàn thành bài tập
Khi đánh dấu hoàn thành bài tập, điểm thưởng được cộng đúng vào tổng điểm của bé và ghi nhận ngày học liên tục, bên cạnh việc ghi lại sự kiện hoạt động đã có sẵn.

💾 Data Reliability  ·  Độ tin cậy dữ liệu
EN
Points & streaks now saved to the database
Previously the child's total points, daily streak and badges were only stored in the browser's local memory (localStorage). If a family cleared browser data or switched devices, all progress was lost. Now the database is the primary source of truth: on load the app syncs from the server, and every change is written through immediately.
VN
Điểm & chuỗi ngày học được lưu vào cơ sở dữ liệu
Trước đây điểm tổng, chuỗi ngày học liên tục và huy hiệu của bé chỉ lưu trong bộ nhớ cục bộ của trình duyệt. Nếu xóa dữ liệu trình duyệt hoặc đổi thiết bị, toàn bộ tiến trình bị mất. Bây giờ cơ sở dữ liệu là nguồn dữ liệu chính: khi mở app sẽ đồng bộ từ server, mọi thay đổi được ghi ngay lập tức.
✅ Done

✅ Done

Một lỗi ở tầng cơ sở dữ liệu (BUG-009) có thể khiến thao tác sửa/xóa tiết học ảnh hưởng đến dữ liệu của người dùng khác. Đã thêm bộ lọc quyền sở hữu chặt chẽ vào tất cả thao tác ghi lịch.
✅ Done

🏗️ Code Architecture  ·  Kiến trúc mã nguồn Dev-facing
EN
Layering violations resolved across 7 modules
Server actions were bypassing the service layer and directly querying the database. This has been corrected — 3 new service files created, 2 extended, and 7 action files updated so data always flows: Action → Service → Repository. A shared requireParentSession guard was also extracted to server/lib/auth-guard.ts.
VN
Sửa vi phạm phân tầng trong 7 module
Các server action đang bỏ qua tầng service và truy vấn trực tiếp vào cơ sở dữ liệu. Đã khắc phục — tạo 3 file service mới, mở rộng 2 file, cập nhật 7 action để dữ liệu luôn đi theo luồng: Action → Service → Repository. Bộ bảo vệ phiên phụ huynh dùng chung cũng được tách riêng ra server/lib/auth-guard.ts.
✅ Done
EN
Parent settings validation enforced
Kid access toggle settings are now validated with a strict schema (Zod) before being saved, preventing malformed data from reaching the database.
VN
Xác thực cài đặt quyền truy cập của bé
Các cài đặt bật/tắt quyền truy cập của bé được xác thực bằng schema nghiêm ngặt (Zod) trước khi lưu, ngăn dữ liệu sai định dạng vào cơ sở dữ liệu.
✅ Done
