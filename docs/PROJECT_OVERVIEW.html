2.3 Technical Architecture at Runtime

Browser UI -> App Router page/layout render -> Server Actions (validated by Zod) -> Service layer (business logic) -> Repository layer (Prisma queries) -> PostgreSQL Cross-cutting runtime guards: - middleware.ts (kid/parent session verification + parent auth rate limit) - ErrorBoundary wrappers in dashboard/games layout groups - Activity logging repository for selected user actions

4.2 Current Limitations

No automatic drift detection

There is no script that checks whether route UI and design JSX are out of sync; compliance is review-based.

No CI enforcement

No GitHub workflow currently validates checklist completion, token usage policy, or route coverage against design inventory.

No machine-readable spec index

Route/design status is Markdown only; no JSON manifest exists to power automated checks.

Prompt correctness depends on humans

Templates are strong but not executable policies; agents can still skip steps without explicit guardrails.

4.3 Practical Automation Plan (High ROI)

AutomationWhat to BuildImpact
