English Route — Design Specification

Source design: design/pages/Kid Hub English - Responsive.html · Components: design/components/english.jsx · Current implementation: components/games/EnglishHub.tsx · Responsive rules: docs/FE/RESPONSIVE_SUMMARY.md

Responsive convention (RESPONSIVE_SUMMARY §1): Base styles target P1 landscape (short height, usable width). Portrait overrides use the portrait: variant. Sidebar/desktop density uses lg: (≥ 1024 px). Kid interactive targets ≥ min-h-tap (48 px); primary actions ≥ min-h-tap-lg.

1. Scope

This document covers the /english hub screen only — the three game-type cards (Alphabet Explorer, Word Safari, Sound Hunt) and the homework banner. The individual mini-game screens (AlphabetGame, WordSafariGame, SoundHuntGame) share the GameShell pattern and are not part of this redesign.

The design file renders five viewports × four sections (Hub + 3 mini-games). This document focuses on the Hub section.

5. Responsive Tailwind Strategy

The design calls for two different card shapes based on orientation, not just size. This means the EnglishHub component needs to conditionally render the right card type, or use a single card that switches its internal layout via portrait:. The recommended approach is two named sub-components to keep each variant readable and testable.

G-10 (RESPONSIVE_SUMMARY §9.2): The existing EnglishHub uses overflow-y-auto on the full-height hub. The new design also scrolls in portrait. This is acceptable for the hub menu (not the game canvas). Confirm with PM and note it in the PR per the gap resolution plan.
