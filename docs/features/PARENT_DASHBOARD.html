1. Current State — What's Already Built

The parent area covers 4 routes. Below is what each one currently does and its fidelity to the design.

/parent ✓ Implemented app/(parent)/parent/page.tsx + ParentDashboardView.tsx

Two-panel dashboard (or tabbed on phone). Data fetched via getScheduleAction and getReportCardAction as Server Component props.

📅 Schedule Manager
Full CRUD on school periods and extra classes. Add/edit/delete periods, cancel a specific occurrence with override.
Actions: createPeriod · updatePeriod · deletePeriod · createExtraClass · cancelExtraClass
🌟 Grades Manager
Parent edits each subject's score for S1/S2. Badge tier (excellent/good/needs-practice) auto-calculated from score.
Action: upsertGradeAction (parent-auth guarded)
📝 Homework Add
Partial
Parent can add/delete daily homework items only through the Schedule Manager row. There is NO read-only parent view of homework status or completion rate.
Actions exist but only used kid-side for marking done
/parent/kid-access ⚠ Partial — static data components/parent/kid-access/KidAccessView.tsx

Controls what the kid can access. Works for feature toggles. Screen time and recent activity are entirely hardcoded.

🛡️ Feature Toggles
Parent can turn on/off individual features (games, views, settings) per group. Stored in localStorage via useLocalStorage.
Note: persisted in browser storage only — not synced to DB
🔑 Kid Pattern Setup
Parent can set the kid's unlock pattern. Backed by server action. Works correctly.
Stored server-side correctly

HARDCODED
Shows "47 / 120 min" always. Hardcoded in KidAccessView.tsx lines 24-25. No DB model for screen time exists.

📋 Recent Activity Feed
HARDCODED
3 static items. No activity logging exists in the DB schema or repository layer. Would need a new model.

/parent/pin ✓ Implemented

PIN entry with lockout logic, shake animation, session cookie set on success. Fully matches design.

3. Proposed Additions

G1 — Homework Status Panel (new tab in /parent) P0

Add a third tab "📝 Bài tập" alongside Lịch học and Điểm số. The tab shows today's homework list and which ones the kid has checked off.

  • Re-use getTodayHomeworkAction in ParentDashboardPage (already no-auth, safe to call server-side).
  • Display: subject pill + homework label + ✅/⬜ done indicator + timestamp when completed.
  • Parent can also delete a homework item from this tab via deleteDailyHomeworkAction.
  • Show completion summary badge: "3/5 hoàn thành".

G2 — Kid Progress & Stats Panel (new section in /parent/kid-access) P1

Replace the hardcoded recent activity sidebar with a real Kid Stats Card that shows points, streak, and game best scores pulled from UserProgress.

  • New read action needed: getKidProgressAction in server/actions/progress.actions.ts (parent-auth guarded, reads UserProgress + GameBestScore[]).
  • New read in progress.repository: getFullProgress(userId) — joins UserProgress with GameBestScore and EarnedBadge.
  • Display: total points · current streak · math/english best star scores · badges earned.
  • The existing ActivitySidebar in KidAccessView.tsx can be replaced with this component.

G3 — Real Screen Time Tracking P2

Track actual time spent in the kid session and compare against a parent-configurable daily limit.

  • New Prisma model: ScreenTimeLog { id, userId, date, startedAt, endedAt, durationSecs }.
  • Kid layout (app/(dashboard)/layout.tsx) starts a session on mount and closes it on unload via Server Action.
  • New parent action: getScreenTimeAction(date) — sums durations for userId+date.
  • New parent action: setScreenTimeLimitAction(limitMinutes) — stores on User.screenTimeLimitMins (new field).
  • Parent /parent/kid-access screen time card becomes live: shows real used/limit with an edit button for the limit.
⬤ High effort — DB migration required; session tracking needs careful edge-case handling (browser close, background)

G4 — Real Activity Feed P2

Log key kid-side events (game finished, page visited, homework marked) and surface the last 10 in /parent/kid-access.

  • New Prisma model: ActivityEvent { id, userId, type, label, createdAt }. Types: GAME_COMPLETE | PAGE_VIEW | HOMEWORK_DONE.
  • Emit events at the end of game sessions (already have saveMathProgressAction — add a log call there).
  • New parent action: getRecentActivityAction(limit).
  • Replace static RECENT_ACTIVITY array in KidAccessView.tsx with server-fetched data.
⬤ High effort — DB migration + event emission in multiple existing actions

G5 — Persist Feature Toggles to DB P1

Currently, kid access toggles only live in localStorage. On a new device or after clearing browser data, all restrictions reset to default (everything ON).

  • New JSON column on User model: kidAccessSettings Json?.
  • New actions: getKidAccessSettingsAction + saveKidAccessSettingsAction(settings) (parent-auth guarded).
  • KidAccessView migrates from useLocalStorage to a new hook useKidAccessSettings that calls these actions.
  • Kid-facing middleware/layout reads the DB value to enforce toggle restrictions (currently toggles are parent-UI-only and not actually enforced).
⬤ Medium effort — DB migration + 2 actions + hook refactor + enforcement layer

G6 — "Today at a Glance" summary view in /parent P1

Add a new default landing panel (or a new tab "🏠 Hôm nay") that gives parent a read-only summary instead of dropping them into the schedule editor.

  • Reuse getTodayViewAction (already returns school periods + evening blocks + homework for today).
  • Combine with G2 kid stats — total points + streak at the top.
  • Three columns on tablet: Today's Classes · Homework Status · Kid Stats.
  • Navigation: "Hôm nay" becomes the default tab; "Lịch học" and "Điểm số" move to an "Chỉnh sửa" sub-section.
