Schedule Route — Design Spec

Source design: design/components/schedule.jsx · Target route: app/(dashboard)/schedule/ · Status: Pending Review

1. Summary of Changes

The current schedule is a single 5-column grid that works acceptably on tablet landscape but is unusable on phone and cramped on phone landscape. The redesign introduces orientation-aware rendering: a tab-per-day list view on phone portrait, a compact mini-grid on phone landscape, and a richer grid + panel layout on tablet and desktop.

The largest breaking change is in subject data: the new PeriodCell uses color-mix(in oklab, <hex> 14%, white) for cell tinting, which requires each subject to expose a raw hex color rather than a Tailwind class name.

Schedule is inside (dashboard) so AppSidebar is already provided by app/(dashboard)/layout.tsx. The hub components do NOT need to import it themselves (unlike /math and /english which are in the separate (games) layout).

3. Grid Orientation System

orient="rows-periods" (Portrait)

4. PeriodCell Redesign

This is the most visually significant change. The current cell is a passive div; the new cell is a tappable button with subject-color tinting.

PropertyCurrentNew
Element div button (interactive, calls onPick)
Normal bg Full subject color via colorClass (e.g. bg-blue-500) color-mix(in oklab, <hex> 14%, white) — very light tint
Current (isNow) bg Same color + scale-[1.03] shadow-lg ring-2 ring-white Full subject <hex> + box-shadow: 0 10px 24px -12px <color>
Text color Always text-white Normal: subject hex; isNow: #fff
Content Subject name + time (time hidden in portrait) Subject icon (emoji) + name; compact hides time, mini = icon+name only on one line
Pulse dot White ping animation (top-right) Same — white dot top-right when isNow
Free cell bg-slate-50, — text bg-#f8fafc, border: 2px dashed #e2e8f0, — text
Border radius rounded-xl (12px) Normal: 14px; compact: 10px (via inline style)
Breaking dep: color-mix(in oklab, …) requires a CSS color value, not a Tailwind class name. Subject objects must gain a color: string field (hex). See Delta D-01.

10. Design Tokens Required

--color-schedule-deep #1d4ed8 Add to @theme Today tab inactive text, today column header text

These token names mirror the --color-math-* / --color-english-* naming pattern already in app/globals.css. The schedule accent is intentionally the same blue as math (both use #3b82f6) — different semantic token, same value.

12. TodayPlanCard — Disposition

Decision needed: The current page shows TodayPlanCard (evening classes + homework) in the right panel. The new design replaces the right panel with a "Hôm nay" school timetable summary (accent card + SubjectLegend). The evening plan is no longer in the design mockup.

Options:
  1. Remove TodayPlanCard from the schedule page entirely — its content (evening homework) is visible elsewhere (dashboard).
  2. Keep it below the SubjectLegend in the aside (may crowd the panel).
  3. Move it to a separate section of phone portrait (below DayList).
Recommendation: Option 1 — the schedule route focuses on school timetable only. Evening plan stays on the dashboard.

14. Responsive Layout Structure

Following the same pattern as EnglishHub and MathHub — both DOM trees in the page, CSS hides the wrong one:

// app/(dashboard)/schedule/page.tsx (simplified structure)
<div className="h-dvh bg-shell-kid overflow-hidden portrait:overflow-y-auto">

  {/* ── PHONE PORTRAIT ─────────────────────────────────────────── */}
  <div className="flex portrait:flex hidden flex-col h-full px-3.5 pb-4 gap-3">
    <Header />                  {/* "Lịch học" + week info + edit btn */}
    <DayTabs ... compact />
    <DaySummaryCard />          {/* period count + day name + time range */}
    <DayList ... />             {/* scrollable period rows */}
  </div>

  {/* ── LANDSCAPE: tablet-portrait / tablet-L / desktop ────────── */}
  <div className="portrait:hidden flex h-full gap-4 p-4 lg:p-6">
    <div className="flex-1 flex flex-col gap-3 min-w-0">
      <Header />
      {/* Phone landscape: rows-periods compact+mini */}
      {/* Tablet portrait: rows-periods full */}
      {/* Tablet-L / Desktop: cols-periods full */}
      <WeekGrid orient={...} ... />
      {/* Tablet portrait bottom rail */}
      <TodayRail className="hidden lg:hidden ..." />
    </div>
    {/* Tablet-L + Desktop right aside */}
    <aside className="hidden lg:flex flex-col gap-3 w-[280px] lg:w-[320px]">
      <TodayCard />             {/* accent bg, today's periods */}
      <SubjectLegend />
      {/* Desktop only: selected period detail + week stats */}
    </aside>
  </div>

</div>
Note: the (dashboard) layout already applies pl-16 portrait:pl-0 lg:pl-56 portrait:pb-16 to the page container (from app/(dashboard)/layout.tsx). The schedule page itself does not need to re-apply sidebar offset classes — this differs from /math and /english which are in the sidebar-less (games) layout.
