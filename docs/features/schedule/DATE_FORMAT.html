1. Overview & Goal

Currently every day label in the schedule shows only the day name ("Thứ Hai", "Hai", etc.). There is no way to tell which actual date a column or tab represents without checking the week subtitle. This spec adds a DD/MM date below or beside every day label — in the main schedule grid, day tabs, summary cards, and the parent schedule manager — so the parent and kid always know exactly which date they are looking at.

2. New Utility Function — getWeekDates()

A single pure function added to lib/schedule-display.ts. It wraps the existing getMondayForWeekOffset() to produce a Record<DayOfWeek, string> mapping each day to its "DD/MM" label.

/** Returns DD/MM date string for every day of the week at the given offset.
 *  weekOffset 0 = current week, -1 = last week, +1 = next week. */
export const getWeekDates = (weekOffset: number): Record<DayOfWeek, string> => {
  const monday = getMondayForWeekOffset(weekOffset)
  const WEEK_DAYS: DayOfWeek[] = [
    'monday', 'tuesday', 'wednesday', 'thursday',
    'friday', 'saturday', 'sunday',
  ]
  return Object.fromEntries(
    WEEK_DAYS.map((dow, i) => {
      const d = new Date(monday)
      d.setDate(monday.getDate() + i)
      const dd = String(d.getDate()).padStart(2, '0')
      const mm = String(d.getMonth() + 1).padStart(2, '0')
      return [dow, `${dd}/${mm}`]
    })
  ) as Record<DayOfWeek, string>
}

For /dashboard (no weekOffset), today's date is computed directly:

const todayDDMM = (): string => {
  const d = new Date()
  return `${String(d.getDate()).padStart(2,'0')}/${String(d.getMonth()+1).padStart(2,'0')}`
}

This could also be added to lib/schedule-display.ts as getTodayDDMM().

6. Implementation Order

  • Add getWeekDates() and getTodayDDMM() to lib/schedule-display.ts. These are pure functions; verify with a quick unit-test mental check for week offset 0, +1, -1.
  • Update WeekGrid — add dateByDay? prop and render date in both rows-periods and cols-periods header cells. Run the app to visually verify the grid looks correct in both orientations.
  • Update DayTabs — add dateByDay? prop and render date below short name. Check tab height doesn't break the phone portrait layout.
  • Update ScheduleView — compute weekDates from weekOffset, pass to all child components (WeekGrid, DayTabs, DaySummaryCard, TodayAccentCard, WeekEveningSection). Verify prev/next week navigation updates the dates correctly.
  • Update DashboardView — add today's DD/MM to the subtitle after mount. Use suppressHydrationWarning (already present on that element).
  • Update ScheduleManager — add weekDates? prop; thread dates into both day-tab sections.
  • Update ParentDashboardView — compute weekDates and pass to ScheduleManager. Verify that navigating to prev/next week updates the schedule tabs correctly and the read-only banner still works.
  • Run npx tsc --noEmit to verify all prop types are satisfied.
Components updated in this order ensure no "prop not yet available" TypeScript errors during development.

Start from the utility (Step 1) → leaf components (Steps 2–3) → container that wires them (Step 4) → independent route (Step 5) → parent path (Steps 6–7).
