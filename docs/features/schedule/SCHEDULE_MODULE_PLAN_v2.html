4. Product Quality Gaps

P0 Evening classes invisible to kid
Parent can create EXTRA_CLASS entries in ScheduleManager, but no kid route renders them. Trọng Khôi cannot see English Tue/Thu 18:00–19:30 anywhere in the app.
P0 Homework completion does not award points
The reward loop (toggleHomeworkDoneAction → awardPointsAction) exists in HomeworkCheckbox but the live homework routes use the legacy markHomeworkDoneAction path with no points field.
P1 Cancelled extra class behaviour contradicts PM decision
buildTodayView calls filterCancelledSlots which removes cancelled classes. PM decision #4: keep visible with strikethrough + "Đã huỷ" badge. EveningBlockChip supports strikethrough but never receives cancelled items.
P1 Orphaned components (dead code risk)
TodayPlanCard, HomeworkCheckbox, EveningBlockChip are fully built but unused. Will drift from live design tokens and subject data unless wired or deleted.
P1 Dashboard homework panel is read-only
DashboardView shows homework items with static check circles — no tap handler, no navigation to /homework, no points display.
P2 Time-band semantic tokens missing
v1 specified --color-band-morning/afternoon/evening in @theme {}. Current evening/homework components use raw palette classes (violet-50, amber-50, emerald-50).
P2 No Playwright e2e coverage
No specs under e2e/schedule/. Responsive schedule redesign and homework reward loop are untested in CI.
OK Parent configuration flow
ScheduleManager 3-tab UI, evening cap validation, daily homework quick-add, PIN-gated mutations — all functional.

5. Recommended Next Steps

PM must choose a disposition for evening + homework before dev continues. Three viable paths:

Option A — Wire TodayPlanCard into existing surfaces (lowest churn)

  • Add getTodayViewAction fetch to /dashboard and render TodayPlanCard below the day rail (landscape) or as a new section (portrait).
  • Replace HomeworkItemRow on /homework with HomeworkCheckbox (or merge points logic into HomeworkItemRow).
  • Fix buildTodayView: stop filtering cancelled IDs; pass full evening list + cancelledIds to EveningBlockChip.
  • Keep /schedule as school-timetable-only per SCHEDULE_DESIGN Option 1.
Effort: Low–Medium · ~2–3 files wired, 1 service fix

Option B — Integrate evening into ScheduleView (matches v1 two-panel vision)

  • Restore two-panel landscape layout: WeekGrid + TodayPlanCard side-by-side.
  • Phone portrait: add "Hôm Nay" tab with TodayPlanCard content below DayList.
  • Requires reconciling aside panel crowding noted in SCHEDULE_DESIGN §12.
Effort: Medium · ScheduleView refactor + responsive testing

Option C — Delete orphaned components; rebuild homework rewards in /homework only

  • Remove TodayPlanCard, HomeworkCheckbox, EveningBlockChip.
  • Enhance /homework with points + star animation; add evening section to dashboard or new /today route.
  • Higher design work; avoids maintaining two homework UIs.
Effort: High · design + delete + rebuild
Recommendation: Option A aligns with the already-shipped SCHEDULE_DESIGN Option 1 (schedule = school only) while completing the v1 feature promise on the dashboard and homework routes. Fix the cancelled-class service bug regardless of which option is chosen.
